# Changelog

Changelog entries follow [Keep a Changelog](https://keepachangelog.com/en/1.0.0/) and [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## Adding Entries

1. Edit `unreleased.md`
2. Add the entry under the appropriate section:
   - **Added** – New features
   - **Changed** – Changes to existing functionality
   - **Deprecated** – Soon-to-be removed features
   - **Removed** – Removed features
   - **Fixed** – Bug fixes
   - **Security** – Security-related changes
3. Keep entries concise and user-focused
4. Run `make changelog` to regenerate the root `CHANGELOG.md`
